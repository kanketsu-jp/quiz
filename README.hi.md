# quiz

> 🇺🇸 [English](./README.md) | 🇯🇵 [日本語](./README.ja.md) | 🇨🇳 [简体中文](./README.zh-Hans.md) | 🇪🇸 [Español](./README.es.md) | 🇮🇳 **हिन्दी** | 🇧🇷 [Português](./README.pt.md) | 🇮🇩 [Bahasa Indonesia](./README.id.md)

Claude Code के लिए एक इंटरैक्टिव कोडिंग क्विज़ स्किल। विकास के दौरान सीखी गई बातों को — बग, लाइब्रेरी, डिबगिंग अंतर्दृष्टि — क्विज़ प्रारूप में दोहराएँ और मजबूत करें।

## इंस्टॉल

```
/plugin marketplace add kanketsu-jp/quiz
/plugin install quiz@kanketsu-quiz
```

## उपयोग

### 1. अध्ययन सामग्री बनाएँ

`.temp/learn/{संख्या}/` डायरेक्टरी में दो फ़ाइलें बनाएँ:

```
.temp/learn/1/
├── study-notes.md   # अध्ययन नोट्स (संदर्भ, कोड, व्याख्या)
└── qa-list.md       # प्रश्नोत्तर सूची (क्विज़ डेटा स्रोत)
```

या Claude Code से कहें:

```
अभी जो ठीक किया उसे .temp/learn/1/ में अध्ययन सामग्री के रूप में सारांशित करो।
study-notes.md और qa-list.md बनाओ।
```

### 2. क्विज़ चलाएँ

```
/quiz 1            # क्विज़ #1 चलाएँ
/quiz scrollarea   # कीवर्ड से खोजें
/quiz              # उपलब्ध क्विज़ सूचीबद्ध करें
/quiz list         # उपलब्ध क्विज़ सूचीबद्ध करें
```

## क्विज़ मोड

| मोड | विवरण |
|-----|-------|
| सभी प्रश्न | सभी प्रश्न क्रम में |
| रैंडम 5 | 5 यादृच्छिक प्रश्न |
| श्रेणी के अनुसार | श्रेणी चुनें |

## फ़ाइल प्रारूप

### qa-list.md

```markdown
## श्रेणी नाम

### Q1: प्रश्न पाठ?

#### A

उत्तर पाठ (कोड ब्लॉक या सादा पाठ)
```

`##` शीर्षक श्रेणियाँ हैं। `### Q:` प्रश्न है। `#### A` उत्तर है।

### study-notes.md

मुक्त प्रारूप Markdown। पृष्ठभूमि, मूल कारण विश्लेषण, सुधार कोड, सबक। गलत उत्तर पर व्याख्या के लिए संदर्भित।

## सर्वोत्तम अभ्यास

| कब | क्या करें |
|----|----------|
| बग ठीक करने के बाद | Claude से अध्ययन सामग्री बनाने को कहें |
| अगले दिन | `/quiz` से कल की सीख दोहराएँ |
| सप्ताहांत | `/quiz` से साप्ताहिक समीक्षा |
| कोड समीक्षा के बाद | समीक्षा प्रतिक्रिया को सामग्री में बदलें |

## .gitignore

`.temp/` को `.gitignore` में जोड़ना चाहिए। अध्ययन सामग्री केवल स्थानीय है।

```gitignore
.temp/
```

## कॉन्फ़िगरेशन

डिफ़ॉल्ट रूप से, quiz `.temp/learn/` से अध्ययन सामग्री पढ़ता है। कॉन्फ़िगरेशन फ़ाइल से पथ कस्टमाइज़ कर सकते हैं:

| प्राथमिकता | फ़ाइल | प्रारूप |
|------------|-------|---------|
| 1 | `.claude/quiz.json` | `{ "learn_dir": "path/to/dir" }` |
| 2 | `.claude/settings.local.json` | `{ "quiz": { "learn_dir": "path/to/dir" } }` |
| 3 | `.claude/settings.json` | `{ "quiz": { "learn_dir": "path/to/dir" } }` |
| 4 | डिफ़ॉल्ट | `.temp/learn/` |

`.claude/quiz.json` उदाहरण:

```json
{
  "learn_dir": "docs/quiz-materials"
}
```

## सुरक्षा

- केवल `.claude/` डायरेक्टरी से कॉन्फ़िगरेशन फ़ाइलें पढ़ता है (`learn_dir` रिज़ॉल्यूशन के लिए)
- केवल कॉन्फ़िगर किए गए डायरेक्टरी से अध्ययन सामग्री फ़ाइलें पढ़ता है
- कोई शेल कमांड नहीं चलाता
- कोई फ़ाइल संशोधित नहीं करता
- अध्ययन सामग्री में निर्देश अनदेखा किए जाते हैं (केवल सामग्री के रूप में माने जाते हैं)
